

## Добавление переключателя "Превью" в библиотеку

### Концепция

В каждом разделе библиотеки добавляется toggle "Превью". Когда включён — при клике на материал сначала показывается короткий отрывок (превью-режим с ограничением по времени и кнопкой "Смотреть полностью"). Когда выключён — видео открывается сразу целиком. Состояние сохраняется в `localStorage` per user.

### Изменения

**1. `src/pages/LibraryPage.tsx`** — добавить toggle "Превью"
- Импортировать `Switch` из UI
- Состояние `previewEnabled` читается из `localStorage` (ключ: `preview_${mockUser.id}`)
- При переключении — сохраняется в `localStorage`
- Toggle размещается справа от заголовка "Библиотека" в одной строке
- Передаём `previewEnabled` в `MaterialCard` как prop

**2. `src/components/MaterialCard.tsx`** — принимать prop `previewEnabled`
- Если превью включено: ссылка ведёт на `/material/${id}?preview=1`
- Если выключено: ссылка ведёт на `/material/${id}` (как сейчас)

**3. `src/pages/MaterialPage.tsx`** — реализовать превью-режим
- Читать query-параметр `preview` из URL
- Если `preview=1`:
  - Для видео с `video_url`: показать iframe но с оверлеем, который через 15 секунд (или сразу) предлагает кнопку "Смотреть полностью" (убирает `?preview`)
  - Для материалов без URL / аудио: показать placeholder с кнопкой "Слушать полностью"
  - Визуально: полупрозрачный gradient-оверлей снизу с кнопкой и текстом "Это превью — нажмите для полного просмотра"
- Если без `preview`: текущее поведение без изменений

### Файлы

| `src/components/MaterialCard.tsx` | Новый prop `previewEnabled`, условная ссылка |
| `src/pages/MaterialPage.tsx` | Превью-режим с оверлеем и кнопкой "Смотреть полностью" |
