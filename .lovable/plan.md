

## Исправление бага «437 из 180 дней»

### Проблема

Логика прогресса берёт `totalDays` (все дни с регистрации) и показывает «X из Y дней», где Y — дни до **следующего** статуса. Если пользователь давно зарегистрирован, X может превышать Y, что выглядит абсурдно (437 из 180).

Кроме того, `targetDays` считается как `nextM.months * 30`, хотя должен показывать дни **текущего** milestone (того, к которому пользователь идёт).

### Исправление

**`src/components/AmbassadorTimeline.tsx`** — изменить логику `progressText`:

1. Показывать прогресс только для **текущего** (ещё не достигнутого) milestone, а не для уже достигнутого.
2. `targetDays` = `milestone.months * 30` для текущего milestone.
3. `displayDays` = `Math.min(totalDays, targetDays)` — не превышать целевое значение.
4. Если `totalDays >= targetDays` — не показывать текст прогресса вообще (статус достигнут, должна быть галочка).

Конкретно: заменить условие `isCurrent && nextMilestoneIdx >= 0` на проверку для первого **недостигнутого** milestone, и использовать его собственный `months * 30` как цель.
